산불 확산 예측 시스템
DEM 격자 셀룰라 오토마타 엔진 v3 · Rothermel(1972) 기반
목차
① 시스템 개요 ② 초기확산 예측지도 사용법 ③ 전략상황판 사용법 ④ GPS 전송 사용법 ⑤ 계산 이론 및 참고논문 ⑥ 주의사항
① 시스템 개요
본 시스템은 산불 초기 대응 시 현장 지휘관이 스마트폰 또는 태블릿으로 신속하게 확산 예측 및 자원 배치를 할 수 있도록 설계된 독립형(Standalone) HTML 애플리케이션입니다.

외부 서버 없이 단일 HTML 파일로 작동하며, GPS 연동 시에만 Firebase Realtime Database를 사용합니다.
② 초기확산 예측지도 — 지휘관 사용 순서
1
📍 발화 위치 입력
📡 현위치 버튼을 누르면 GPS로 자동 입력됩니다.
직접 입력 시: 위도(예 35.8714) / 경도(예 128.6014) 순으로 입력하세요.
※ 네이버·카카오 지도에서 발화 지점을 길게 누르면 좌표 확인 가능
2
🧭 풍향 선택
불이 번지는 방향을 8방위 버튼에서 선택합니다.
예) 서풍이 불어 불이 동쪽으로 번진다 → 동 선택
※ 현장 기상청 예보 또는 현장 체감 풍향 기준
3
💨 풍속 · 📐 경사도 · 💧 상대습도 입력
풍속 슬라이더로 현재 풍속(km/h) 설정
기상청 예보 m/s × 3.6 = km/h 환산
경사도 발화 지점 주변 산의 평균 경사각(°)
완만한 야산 10~15° / 급경사 30° 이상
상대습도 현재 대기 습도(%)
건조특보 발령 시 20~30% 이하 입력
4
🌲 수종 · ⏱ 경과 시간 선택
수종: 발화 지점 주요 수종 선택
대구·경북 대부분 → 소나무 / 해안 → 곰솔 선택
경과 시간: 몇 분 후의 확산 범위를 예측할지 선택
초기 대응 시 30분, 광역 대응 시 1~3시간 권장
5
🔥 지도에 확산 범위 표시 버튼 클릭
빨간 실행 버튼을 누르면 DEM 격자 계산이 시작됩니다.
수초 내에 지도와 함께 확산 범위가 표시됩니다.
6
📊 결과 탭 확인 및 방어선 설정
불머리 거리 바람 방향으로 확산되는 최대 거리(m)
→ 이 거리 앞에 방어선 선점 필요
피해 면적 예상 연소 면적(ha)
축구장 1개 ≈ 0.7ha 참고
확산 속도 평균 ROS (m/분)
12 이상 → 극도 위험, 즉시 대피 검토
방어선 좌표 불머리 전방 방어선 권고 위치
복사 버튼으로 좌표 공유 가능
지도 색상 읽는 법
빨강 = 초기 연소 (중심부) 주황 = 중간 확산 노랑 = 화선 최외곽
③ 전략상황판
OpenStreetMap 전체화면 지도 위에 드로잉 보드를 제공합니다. 현장 상황을 지도에 직접 그려 공유할 수 있습니다.
🛠️ 도구
🖐 이동 지도 드래그 이동 모드
✏️ 연필 자유 선 그리기
🩹 지우개 부분 지우기
🪣 칠하기 영역 채우기
🚒 소방 자원 아이콘
소방펌프차 · 구급차 · 고가사다리차 · 소방지휘차 · 소방탱크차 · 소방관 · 소화전 · 25㎜/40㎜/65㎜ 호스 · 호스릴
크기 슬라이더로 20~80px 조절 가능
📌 아이콘 배치
1. 원하는 아이콘 클릭 → 테두리 노란색으로 강조
2. 지도 위 원하는 위치 클릭 → 아이콘 + 라벨 배치
⬚ 아이콘 선택 · 이동 · 삭제
1. ⬚ 선택 도구 클릭
2. 배치된 아이콘 클릭 → 노란 점선 테두리로 선택됨
3. 선택된 아이콘을 드래그 → 원하는 위치로 이동
4. 하단 액션바 🗑 삭제 버튼 → 해당 아이콘 제거
5. 빈 곳 클릭 또는 ✕ 취소 → 선택 해제
✏️ 라벨(이름) 수정
방법 1 — 더블탭
⬚ 선택 도구 선택 → 아이콘을 빠르게 두 번 탭
→ 라벨 편집창 팝업 → 수정 후 ✓ 확인

방법 2 — 액션바 버튼
아이콘 클릭(선택) → 하단 액션바 ✏ 라벨 버튼
→ 라벨 편집창 팝업 → 수정 후 ✓ 확인

※ Enter 키로 확인, Escape 키로 취소 · 최대 12글자
📡 GPS 마커 수신
지도 통제권은 지휘관에게 있습니다.

① 지도 초기 위치
2번 전략상황판 진입 시 지휘관의 GPS 위치를 기준으로 지도가 초기화됩니다.
이후 지도 이동·확대·축소는 🖐 손바닥 도구로 지휘관이 직접 조작합니다.

② 대원 위치 표시
4번 GPS 전송 탭에서 이름을 입력하고 전송한 대원의 위치가 지도에 파란 핀 마커 + 이름 배지로 표시됩니다.
대원의 GPS 값은 표시만 될 뿐 지도 위치·줌을 변경하지 않습니다.

③ 전송 순서 무관
전략상황판을 먼저 켜도, 대원이 먼저 전송해도 동일하게 작동합니다.
Firebase가 기존 데이터를 보관하므로 접속 즉시 모든 대원 위치가 표시됩니다.
🖥 화면 공유 (지휘관 ↔ 대원)
역할 선택
2번 전략상황판을 누르면 역할 선택 화면이 나타납니다.
✏️ 지휘관 — 드로잉·아이콘 배치·화면 공유 업로드
👁 대원 (보기) — 지휘관 화면 실시간 수신만
✏️ 지휘관 모드
상단에 ✏️ 지휘관 배지 표시
드로잉·아이콘 배치 완료 후 3초 뒤 자동으로 Firebase에 화면 업로드
별도로 공유 버튼을 누를 필요 없이 자동 처리됩니다.
👁 대원 모드
상단에 👁 대원 배지 표시
드로잉 도구 비활성화 — 지도 이동만 가능
지휘관이 업로드하면 즉시 화면에 오버레이로 표시
화면 우측 하단에 수신 시각 표시

※ 여러 명이 대원 모드로 동시 접속 가능
※ 지휘관은 1명만 운용 권장 (동시 편집 시 덮어씌워짐)
④ GPS 전송
현장 기기에서 이 탭을 선택해 GPS 위치를 지휘소로 전송합니다.
1. 이름 / 차량명 입력
2. 전송 주기 선택 (5초 · 10초 · 30초 · 1분)
3. 📡 GPS 전송 시작 버튼 클릭
4. 지휘소 전략상황판에 자동으로 마커 표시
5. ⏹ 전송 중지로 언제든 종료
⑤ 계산 이론 및 참고논문
ROS 계산 모델 (논문 근거)
ROS = 0.57 × 풍속보정 × 경사보정 × 습도보정 × 수종계수

풍속보정 = (ws^1.7 × 0.72) + 1
→ Andrews(2012) WAF 지수 1.7 적용 · 0.5m/s 증가마다 1.3배 ①②

경사보정
· 15° 미만 → 1.0
· 15° ~ 30° → 1.5 (표 3-7 기준 1.4 반영)
· 30° 이상 → 2.8 (10° 증가마다 2배 ①)

습도보정 = max(0.3, 1.0 − (RH−30) × 0.018)
→ 침엽수 수분함량 110% 기준 적용 ①

수종계수
· 곰솔 1.10 > 소나무·잣나무 1.00 > 리기다 0.97
· 낙엽송 0.85 > 활엽수 0.40 (표 3-32 발열량 기준 ①)

8방향 CA 확산
→ Moore 방식 · 바람 방향 3셀 Pw 가중 적용 ③
DEM 격자 셀룰라 오토마타
30m × 30m 격자(60×60) 기반 확산 시뮬레이션.
각 셀마다 경사도·사면방향·풍향 정렬도를 개별 계산하여 비대칭 확산 형태를 구현합니다.
색상: 빨강(초기 연소) → 주황 → 노랑(화선 최외곽)
참고논문 및 자료
① 국립산림과학원
산림내 주요시설물지역 산불 피해저감기술 개발
풍속→확산율 계수 120~121p 표 3-33 · 풍속 0.5m/s 증가마다 확산속도 1.3배
경사→확산율 계수 120p 표 3-33 · 경사 10° 증가마다 약 2배 증가
경사 위험 가중치 79~80p 표 3-7 · 15°미만=0.5 / 15~20°=1 / 20~30°=1.4 / 30°이상=1.5
습도→확산 보정 122p · 수분함량 높을수록 확산율 감소, 침엽수 평균 110% 기준
수종별 확산계수 118~119p 표 3-32 · 발열량: 리기다>곰솔>소나무>잣나무>낙엽송
수관화 전이 기준 122p · 풍속 4m/s 이상 시 수관화 전이 위험 급격히 상승
② 채희문·이찬용 (2003)
모형실험에 의한 풍속변화에 따른 산불의 확산속도와 강도분석
풍속→확산율 실험검증 215p Table 1 · 풍속 0.5→1m/s 시 확산속도 약 1.4배 증가 실험 확인
③ 송학수 외 (2013)
산불확산패턴 연구를 위한 시뮬레이션 모델
8방향 확산 모델 102p Fig.1 · Moore 방식 8방향 이웃 셀 확산 모델 채택
풍향 방향 보정 102~103p Fig.3 · 바람 방향 쪽 3개 셀에만 풍속 확률(Pw) 추가 적용
바람 확산 가속 105p Fig.6~8 · 바람 셀수록 연소 도달시간(T) 감소 → 확산 가속 확인
④ Andrews, P.L. (2012, USDA)
RMRS-GTR-266 · Rothermel 모델 풍속 보정계수(WAF) 이론
풍속 보정계수 WAF p.1~5 · 20ft 풍속 → 화염 중간 높이 풍속으로 보정, 지수 1.7 적용
⑥ 주의사항
⚠️ 본 시스템의 예측 결과는 참고용이며 실제 진화 작전의 유일한 근거로 사용해서는 안 됩니다.
⚠️ 실제 산불은 비화·수관화·지형 굴절 등 복잡한 요인에 의해 예측과 다르게 확산될 수 있습니다.
⚠️ 합성 DEM을 사용하므로 실제 지형 데이터와 차이가 있을 수 있습니다.
⚠️ GPS 전송 기능은 인터넷 연결과 Firebase 설정이 필요합니다.
⚠️ 현장 기상 변화 시 즉시 재계산하여 예측을 갱신하세요.
